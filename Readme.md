## Описание

**«Кексобукинг»** — сервис размещения объявлений о поиске и сдаче в аренду недвижимости в центре Токио. <br>
Пользователям предоставляется возможность просматривать уже размещённые объявления и размещать объявления о своей недвижимости.

Учебный проект, созданный для отработки навыков работы с современным JavaScript, интеграции сторонних библиотек (Leaflet, Pristine, noUiSlider), а также взаимодействия с формами и сервером.

Вёрстка и стилизация даны изначально, в проекте реализуется только логика работы на JavaScript.

### Стек технологий

- **JavaScript (ES6+):** - основной язык разработки, модульная архитектура
- **noUiSlider:**- продвинутая валидация форм
- **Leaflet:** - Интерактивная карта с метками
- **Pristine.js:** - валидация формы.
- **Fetch API:** Отправка и получение данных с сервера.

## Функциональность
### Работа с картой
С помощью библиотеки Leaflet реализуется отображение карты с указанием глвной метки (ценр Токио) и 10 меток с объявлениями (данные с сервера).

Успешная загрузка данных об объявлениях с сервера переводит в активное состояние фильтры.
В этом состоянии страница позволяет:
- Просматривать похожие объявления на карте;
- Фильтровать объявления;
- Уточнять подробную информацию об объявлениях, показывая для каждого карточку.

Отрисовка соответствующих выбранным фильтрам меток происходит не чаще, чем раз в полсекунды (устранение дребезга).


### Форма для размещения своего объявления
Фрма для размещения своего объявления становится активной после инициализации карты.

При активном состоянии формы страница позволяет:
- Вносить изменения в форму и отправлять её на сервер;
- Выставлять кординаты, перемещая специальную (главную) метку по карте.

Заполнение всей информации производится на одной странице без промежуточных переходов. Порядок заполнения информации не важен.
Если данные, введённые в форму, не соответствуют ограничениям, форму невозможно отправить на сервер.

#### Валидация
 Для валидации объявления и логики обработки пользовательского ввода полей формы используется библиотека Pristine.

При попытке отправить форму с неправильными данными, отправка не происходит, пользователю показываются ошибки для неверно заполненных полей.
Проверка срабатывает при отправке формы на сервер и не даёт отправть её если есть ошибки.

#### Отправка формы
Если отправка данных прошла успешно показывается соответствующее сообщение.

При успешной отправке формы или её очистке, страница, не перезагружаясь, переходит в состояние, когда:
- все заполненные поля возвращаются в изначальное состояние;
- фильтрация (состояние фильтров и отфильтрованные метки) сбрасывается;
- метка адреса возвращается в исходное положение;
- значение поля адреса корректируется соответственно исходному положению метки;
- карточка с подробной информацией скрывается.

Если при отправке данных произошла ошибка запроса, показывается соответствующее сообщение.
В этом случае вся введённая пользователем информация сохраняется, для возможности отправить форму повторно.
